# FreeClaw 协议 v1.1

**让 AI 直接生成"文件系统"，而不是一段代码。**

FreeClaw 协议定义了一种简单、稳定的方式，使 AI 可以一次性输出多个文件，并被程序自动解析为本地项目结构。

---

## 为什么需要它？

大模型可以写代码，但有一个根本问题：

> ❌ AI 输出的是"文本"
> ❌ 开发者需要的是"文件"

于是你不得不：

* 复制
* 粘贴
* 新建文件
* 重命名
* 重复 N 次

👉 这不是 AI 的问题，是**缺少一个标准协议**。

---

## FreeClaw 做了什么？

FreeClaw 不发明新语法，而是直接利用 Markdown：

* `h2 标题` → 文件路径 + 可选行号区间
* `代码块` → 文件内容

👉 **Markdown 结构 = 文件系统**

---

## 协议格式

### 完整替换（默认）

````markdown
## relative/path/to/file.ext
```language
完整源代码
```
````

### 区间替换（行号替换）

````markdown
## relative/path/to/file.ext[start,end]
```language
替换用的代码
```
````

---

## 区间替换语义

### 区间定义

`[start,end]` 指的是**修改前目标文件**的行号区间（闭区间）。

### 行号规则

1. 行号从 1 开始
2. 处理前必须将换行符统一转换为 LF（`\n`）
3. 行号基于原始文本行，不基于 AST 节点

### 替换行为

`[start,end]` 区间内的所有行被**删除**，替换为代码块中提供的内容。

### 单段限制

每个文件条目只允许一个区间。多个区间应拆分为多个独立条目。

### 无效情况

以下情况视为无效，忽略该条目：
- `start > end`
- 缺少文件内容
- 区间格式错误

## 示例

````markdown
## js/services/extractor.js
```javascript
export function extract() {
    return [];
}
```

## js/ui/toast.js[3,4]
```javascript
    // 修改后的代码
    _timer: null,
```
````

## 核心规则

```
1. 每个文件使用一个 h2 标题表示文件路径（可附带行号区间）
2. 文件路径必须是相对于项目根目录的路径（如 js/services/extractor.js）

3. 文件路径必须是规范路径：
   - 不得以 / 或 ./ 开头
   - 不得包含 ..（禁止路径回退）
   - 必须使用正斜杠 /
   - 不得包含额外描述（如 "a.js (updated)"）

4. 每个 h2 后必须跟一个代码块
5. 解析时允许跳过空行或非结构性内容
6. 若存在多个代码块，仅第一个有效

7. 每个 h2 只对应一个文件
8. 代码块必须是完整源码，不得截断或省略（区间替换时提供替换后的内容）
9. 代码块不能为空

10. 同一响应中允许出现重复路径：
    - 完整替换 + 区间替换可同时存在
    - 同名文件的多个条目均有效，用户可选择保存
    - 解析器应保留所有条目，不做覆盖

11. 语言标记仅用于展示，解析应以文件扩展名为准
```

---

## 解析模型

一个符合协议的解析器应当：

1. 遍历所有 h2 标题
2. 提取标题内容作为文件路径（去除 `[start,end]` 部分）
3. 如果存在 `[start,end]`，记录行号区间
4. 向后查找最近的代码块（`pre > code`）
5. 提取代码内容
6. 写入对应路径的文件

---

## 设计原则

### 1. 结构优先

不依赖字符串匹配，而是依赖 Markdown 语法树。

---

### 2. 语言无关

支持任何文件类型（JSON、CSS、Markdown、配置文件等）。

---

### 3. 零侵入

不在文件内容中嵌入任何协议标记。

---

### 4. AI 原生

完全符合大模型的输出习惯，无需学习新语法。

---

### 5. 人类可读

输出结果既能被程序解析，也能直接阅读和编辑。

---

## 对比方案

| 方案               | 问题             |
| ---------------- | -------------- |
| 注释协议（`// file:`） | JSON / 配置文件不支持 |
| 自定义分隔符           | AI 容易输出错误      |
| JSON结构输出         | 可读性差，不适合长代码    |
| FreeClaw 协议      | 简单、稳定、通用       |

## 适用场景

* AI 代码生成工具
* 浏览器 AI 插件（如 FreeClaw）
* Chat → 项目生成
* 自动代码导出
* LLM 与 IDE 集成

---

## 局限性

* 依赖 Markdown 渲染结构稳定性
* 需要配合合理 Prompt 才能达到最佳效果
* 不适用于二进制文件
